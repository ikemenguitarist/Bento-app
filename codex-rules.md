# codex-rules.md

このプロジェクトは Django ベースです。

ルール:
- まず編集前に plan を出す
- 変更ファイルを最初に列挙する
- migration が必要か明示する
- 既存の命名規則に合わせる
- 不要な依存は追加しない
- fat views を避け、必要なら services.py に分離する
- 新機能には最低限のテスト観点を出す
- 破壊的変更がある場合は先に警告する
- 日本語UIの文言は自然な日本語にする
- 原則として 1機能 1ブランチ で作業する
- 別機能に着手する前に、現在ブランチの変更をコミットまたは退避してから新しいブランチを切る
